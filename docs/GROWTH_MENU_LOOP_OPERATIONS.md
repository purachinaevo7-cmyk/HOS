# HOS Growth Menu Loop 運用・監査仕様

## ゴール

1. 毎朝05:30 JSTに4枚のPowerPoint授業を生成する。
2. 生成後に構造、禁止項目、フォント、プロバイダー、プライバシーを監査する。
3. 監査合格品だけを `outputs/growth-menu/latest.pptx` として保存する。
4. 毎朝06:30 JSTにChatGPTのタスクが教材を通知する。
5. ユーザーは同じチャットへ理解内容、ケース見解、英作文を返す。
6. ひーちゃんは回答を評価し、匿名化した診断のみをGitHubへ記録する。
7. 翌日の生成が最新診断を読み込み、テーマと英文法を調整する。

## 教材構成

- 1枚目：学習マップ。テーマ、重要性、学習目標、前回からつなぐ視点。
- 2枚目：理論授業。定義、3つのポイント、3段階の因果、よくある混同。
- 3枚目：ケース授業。状況、観察事実、理論による解釈、要点。
- 4枚目：英文法授業。型、ルール、誤り例、正解例、文の分解、単語、例文、ミニ練習。

PowerPointには提出用紙、回答欄、完了条件、進捗チェックを入れません。提出方法は通知文にのみ置きます。

## 生成プロバイダー

- OpenAI APIは使用しない。
- 本番はGemini無料枠を1日1回だけ使用する。
- デフォルトモデルはリポジトリ変数 `GEMINI_MODEL`。未設定時は `gemini-2.5-flash`。
- Google検索グラウンディングを使わない。
- OpenAI、有料モデル、mockへの自動切替を行わない。
- `GEMINI_API_KEY` がない、レート制限、無料枠上限、モデル障害の場合は明示的に失敗する。
- 手動の `offline=true` だけは、監査・レイアウト確認用の決定論的教材を生成する。

## 教材シード

`curriculum_seeds.json` に、時点で変わりにくい経営概念と一般化したケースを保持します。Geminiは説明を読みやすく整えますが、ケースの状況・観察事実・要点はシード値で上書きし、企業数値や最近の出来事を創作させません。

- 16:9
- Yu Gothic UI
- タイトル20pt、本文16pt、注釈14pt、締め18pt
- 原則として図形内テキスト
- 編集可能なPPTX
- 4枚

## 評価と翌日反映

### 経営・人事
- 概念理解
- 根拠と結論の接続
- 業界要因と個社要因の切り分け
- 顧客価値とコストの両面
- 財務と人材の接続

### 英語
- 主語と動詞
- 冠詞と単数・複数
- 所有格
- because節
- 前置詞と動詞の型
- 自然なビジネス語彙

GitHubへは生の回答を保存せず、「根拠と結論の接続を強化する」「冠詞の欠落を復習する」など一般化した診断だけを保存します。

## 失敗時

- 生成失敗：古い教材を今日分として出さない。
- 監査失敗：`latest.pptx` をコミットしない。
- 06:30時点で当日分がない：ChatGPTは「本日の教材は未生成・要確認」と通知する。
- API障害時にOpenAI APIへ切り替えない。
